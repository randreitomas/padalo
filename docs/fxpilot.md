# Phase 5 FXPilot Forecasting Engine

## Purpose

FXPilot is a provider-behavior forecasting feature for Padalo's AE-PH remittance demo. It estimates
which of the next seven calendar days has the lowest modeled total send cost for a requested PHP
amount. It does not predict the mid-market foreign-exchange rate, execute a remittance, promise a
provider price, or provide financial advice.

The forecasting layer lives in `packages/forecasting` and has no FastAPI, OpenAI, database, or agent
imports. The agent calls it through one typed adapter in
`packages/agent/src/padalo_agent/tools/mock_forecast.py`; the historical filename remains only to
avoid changing the frozen agent routing surface.

```text
Agent forecast_remittance tool
  -> typed adapter
  -> FxPilotForecastService
  -> provider-scoped Prophet model + calendar features
  -> ForecastResult
  -> frozen ForecastRecommendation response
```

## Synthetic Dataset

`data/synthetic/fxpilot_provider_history.csv` is generated by Python, not hand-authored. It contains
3,882 daily observations: Wise, Remitly, and WorldRemit on the AE-PH corridor from 2023-01-01 through
2026-07-17. The companion `fxpilot_dataset_metadata.json` records the dataset version, seed,
coverage, providers, schema, and generation assumptions.

Every row includes:

| Field                               | Meaning                                                          |
| ----------------------------------- | ---------------------------------------------------------------- |
| `provider`                          | Synthetic provider profile.                                      |
| `corridor`                          | Fixed `AE-PH` demo corridor.                                     |
| `date`, `weekday`, `weekday_number` | Calendar dimensions used for time-series behavior.               |
| `fee_php`                           | Modeled provider fee.                                            |
| `effective_exchange_rate`           | Modeled effective PHP-per-AED rate.                              |
| `is_holiday`                        | Philippine public-holiday flag.                                  |
| `is_promo`                          | Deterministic provider promotion-period flag.                    |
| `is_payday`                         | Philippine payday-window flag.                                   |
| `effective_cost_php_15000`          | Training target: modeled fee plus exchange spread at PHP 15,000. |

The generator has seed `20260717`. It adds weekly provider patterns, scheduled 28-day promotions,
payday effects on the 14th-16th and 29th-31st, Philippine holidays, and seeded normal noise. It is
intentionally a demo model, not evidence of real provider pricing behavior.

Regenerate the versioned artifacts after changing the generator:

```bash
npm run forecast:data
npm run forecast:evaluate
```

Windows PowerShell without activating the API virtual environment:

```powershell
npm run forecast:data:win
npm run forecast:evaluate:win
```

## Prophet Configuration

One Prophet model is fitted per provider against `effective_cost_php_15000`.

| Parameter                  |                   Value | Reason                                                                                                                                                                  |
| -------------------------- | ----------------------: | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Weekly seasonality         |                 enabled | Captures recurring weekday provider behavior.                                                                                                                           |
| Yearly / daily seasonality |                disabled | The synthetic MVP has no justified long-cycle or intraday signal.                                                                                                       |
| Philippine holidays        |         custom calendar | Accounts for New Year, Araw ng Kagitingan, Labor Day, Independence Day, National Heroes Day, Bonifacio Day, Christmas Day, Rizal Day, Maundy Thursday, and Good Friday. |
| Regressors                 | `is_payday`, `is_promo` | Known-in-advance calendar effects.                                                                                                                                      |
| Interval width             |                    0.80 | Produces a practical uncertainty interval without an overconfident presentation.                                                                                        |
| Uncertainty samples        |                     300 | Stable demo intervals at modest cost.                                                                                                                                   |
| Changepoint prior scale    |                    0.08 | Keeps the short MVP history from overreacting to noise.                                                                                                                 |
| Seasonality prior scale    |                     4.0 | Lets the designed weekly pattern remain visible.                                                                                                                        |
| Regressor prior scale      |                     3.0 | Balances promo and payday signals against the weekly component.                                                                                                         |
| Seed                       |                20260717 | Fixes NumPy sampling before training and prediction.                                                                                                                    |

The service caches immutable in-process models by provider and latest history date. For each request,
it builds the known calendar features for the next seven days, predicts cost, scales the PHP 15,000
target to the requested amount, and recommends the minimum projected cost. Estimated savings are the
difference between that day and the median projected day in the same seven-day window.

## Baseline Evaluation

`data/synthetic/fxpilot_backtest.json` contains a fixed 56-day holdout evaluation. Prophet is compared
with a naive baseline that predicts a day from the recent same-weekday history. Both RMSE, MAE, and
MAPE are always retained. The preferred model is selected only from lower MAE; a worse Prophet result
would remain visible in the report and documentation.

| Provider   | Prophet RMSE | Prophet MAE | Prophet MAPE | Baseline RMSE | Baseline MAE | Baseline MAPE | Preferred |
| ---------- | -----------: | ----------: | -----------: | ------------: | -----------: | ------------: | --------- |
| Remitly    |         6.47 |        4.96 |        1.50% |         58.45 |        43.56 |        13.90% | Prophet   |
| Wise       |         8.64 |        7.02 |        2.61% |         50.41 |        39.05 |        15.12% | Prophet   |
| WorldRemit |         9.90 |        8.32 |        2.28% |         60.79 |        47.22 |        14.07% | Prophet   |

These results are favorable because the holdout comes from the same designed synthetic process used
for training. They demonstrate correct evaluation behavior, not real-world forecast accuracy. The
optional `fxpilot_backtest_mae.png` visualizes the MAE comparison.

## Explainability And Contract

`ForecastResult` retains the selected day, expected savings, confidence, provider, amount, human
reasoning, limitations, model version (`fxpilot-prophet-v1`), and synthetic-data provenance. It
explains the recommendation using the historical weekday pattern and states that weekly, payday,
holiday, and scheduled promo effects were considered.

The frozen agent tool contract is deliberately not widened:

```json
{
  "recommended_day": "Thursday",
  "expected_savings_php": "161.96",
  "confidence": "84% historical-pattern confidence",
  "provider": "Wise",
  "amount_php": "15000.00",
  "is_mock": true,
  "disclaimer": "Historical synthetic Wise behavior shows lower modeled total send costs on Thursday..."
}
```

`is_mock` remains `true` because the source is synthetic. It does not mean the adapter returns a
hard-coded recommendation: the value is produced by the Prophet service. The model version and full
reasoning stay inside the forecasting layer because exposing them would change the frozen tool schema.
The disclaimer is mandatory and states that this is not a live quote, provider commitment, or
financial advice.

## Tests

`packages/forecasting/tests` covers deterministic generation, required dataset fields, the Wise
Thursday signal, reported Prophet and baseline metrics, and deterministic provider forecasts.
`apps/api/tests/test_forecast_integration.py` proves that the agent adapter returns a real forecast
without changing the existing `ForecastRecommendation` contract.

Run all forecasting tests with:

```bash
npm run forecast:test
```

## Migration To Real Provider Data

Before FXPilot can be presented as a real provider-behavior estimate, replace only the synthetic-data
input boundary with a consented, licensed, timestamped provider-price feed. Preserve the normalized
columns, validate corridor and currency assumptions, record data freshness and provenance, retrain
and backtest per provider, monitor drift, and raise the confidence threshold only after real holdout
results support it. The agent adapter, tool schema, dashboard SSE events, and card can remain intact.
